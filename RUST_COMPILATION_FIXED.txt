FIX #2: json_handler.rs (merge_json function)

   OLD (Broken):
     fn merge_json(target: &mut Value, source: &Value) {
         match (target, source) {
             // match moves target here
             (Value::Object(...), Value::Object(s)) => { ... }
             (_, _) => *target = source.clone(),  ← ERROR: target already moved
         }
     }

   NEW (Fixed):
     fn merge_json(target: &mut Value, source: &Value) {
         if let Value::Object(ref mut target_map) = target {
             if let Value::Object(source_map) = source {
                 // Use target_map safely
                 for (k, v) in source_map.iter() {
                     merge_json(target_map.entry(k.clone()).or_insert(Value::Null), v);
                 }
                 return;  ← Early return prevents second branch
             }
         }
         *target = source.clone();  ← Safe to use target here
     }

   Why:
     • if let doesn't move the value
     • Early return prevents double-use
     • target_map is a reference, not ownership transfer
     • Second branch only executes if first didn't

💡 KEY LESSONS:
────────────────────────────────────────────────────────────────────────────────

1. Rust Ownership Rules:
   • Can't extend lifetime of temporary values
   • String literals are &str (borrowed)
   • format!() returns String (owned)
   • In vectors, all elements must be same type

2. Borrow Checker:
   • match can move values unexpectedly
   • if let is safer for maintaining borrows
   • Early returns prevent reuse of moved values

3. Collections:
   • Vec expects all elements of same type
   • Convert with .to_string() or format!()
   • Clone to extract from vec
